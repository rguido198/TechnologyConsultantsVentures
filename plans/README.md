## Recommended execution order

1. **001** — highest leverage (real regression on the site's most important card, trivial fix), fully independent.
2. **002** — high-impact performance fix, fully independent. Execute before 003.
3. **003** — depends on 002 (reuses the `IntersectionObserver` and `onScreen` flag it introduces). Do not execute standalone.
4. **004** — independent, moderate effort.
5. **007** — independent, low effort, immediately visible polish gain.
6. **005** — independent; if you're also doing 008, do 005 first so 008 can reference `var(--ease-out)` directly instead of its fallback form.
7. **008** — works standalone (has a built-in fallback for `--ease-out`), but reads more cleanly if 005 has already landed.
8. **006** — requires a human decision on design intent before any code changes; not blocking on anything else, but budget time to actually ask the question rather than defaulting to a guess.

## Dependencies

- **003 → 002**: 003's plan explicitly instructs stopping if 002 hasn't been applied — it extends the same observer callback rather than creating a second one on the same element.
- **008 ⇢ 005** (soft, not blocking): 008 includes a `var(--ease-out, cubic-bezier(0.16, 1, 0.3, 1))` fallback specifically so it works whether or not 005 has run. No re-edit needed if 005 lands afterward.
- All other plans (001, 002, 004, 005, 007) are fully independent and can be executed in any order or in parallel by different agents.
